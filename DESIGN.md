## 2. 不足要件の詳細分析

### GAP-7: マルチエージェント間の共有メモリ

**現状の問題**:
```
現在の協調フロー:
Task → Agent1 → output1 → Agent2(input=output1) → output2 → Agent3 ...
```
- エージェント間は出力を次へ渡すだけで、**共通の知識プール**がない
- あるエージェントが発見した重要な事実を、後続エージェントが参照できない
- セッション終了後、協調作業で得られた知識が永続化されない

**必要な機能**:
- `CooperativeSessionMemory`: セッション内で全エージェントが読み書きできる一時的なキー-バリューストア
- セッション終了時に重要エントリを `LongTermMemory` に昇格させる仕組み

---

### GAP-8: スキル学習フィードバックループ（Hermes Agent パターン）

**Hermes Agent の核心**:
```
実行 → 観察 → 反省（Reflection） → スキル更新 → 次回実行に反映
```

**現状の問題**:
- `MemoryConsolidationService` は「事実」を保存するが「スキル」を自動生成・改善しない
- スキルの有効性を評価する仕組みがない（使われたか？役に立ったか？）
- TriggerKeywords が手動管理のみで、AIが自動更新できない
- 成功したプロンプトパターンから新しいスキルを自動抽出する機能がない

**必要な機能**:
1. スキル使用状況の追跡（`Skill.UseCount`, `Skill.SuccessRate`）
2. タスク完了後の反省フェーズ（Reflection Agent）
3. 新スキル候補の自動提案と承認フロー
4. 既存スキルのキーワード・説明の自動改善

---

### GAP-9: UI の使いやすさ

**問題点**:
- サイドバーに多数の要素が混在（セッション一覧・エージェント・プロジェクト・スキル）
- 初回利用者には「何から始めれば良いか」が不明
- マルチエージェント協調モードの起動方法が直感的でない

## 3. 改善設計

### 3.1 GAP-7: CooperativeSessionMemory の実装

#### 新モデル: `SessionMemory`

```csharp
// Models/SessionMemory.cs
public class SessionMemory
{
    public int Id { get; set; }
    public int ChatSessionId { get; set; }
    public string AgentName { get; set; } = "";      // 書き込んだエージェント
    public string Key { get; set; } = "";             // メモリキー
    public string Value { get; set; } = "";           // メモリ値
    public bool IsPromoted { get; set; } = false;     // LongTermMemoryに昇格済みか
    public DateTime CreatedAt { get; set; } = DateTime.UtcNow;
    
    public ChatSession? ChatSession { get; set; }
}
```

#### 新サービス: `SessionMemoryService`

```csharp
// Services/SessionMemoryService.cs
public class SessionMemoryService
{
    // エージェントがメモリに書き込む
    Task WriteAsync(int sessionId, string agentName, string key, string value);
    
    // 全エージェントからメモリを読む
    Task<string> ReadAllAsContextAsync(int sessionId);
    
    // セッション終了時に重要エントリをLongTermMemoryへ昇格
    Task PromoteToLongTermAsync(int sessionId, int userId);
}
```

#### `CooperateAsync` の変更点

各エージェントのシステムプロンプトに共有メモリの読み取りコンテキストを注入:

```
[共有セッションメモリ]:
- discovery: APIは認証にJWTを使用している
- constraint: レスポンスは500ms以内に返す必要がある
```

各エージェントの出力から構造化情報を抽出してメモリに書き込む:
```
出力解析 → "KEY: value" 形式のメモリエントリを抽出 → SessionMemoryに保存
```

### 3.2 GAP-8: SkillLearningService（Hermes型）

#### Skillモデルの拡張

```csharp
// 既存Skillモデルに追加するフィールド
public int UseCount { get; set; } = 0;           // 使用回数
public int SuccessCount { get; set; } = 0;        // 成功回数（Reviewerが高評価した回数）
public double SuccessRate => UseCount > 0 ? (double)SuccessCount / UseCount : 0;
public DateTime? LastUsedAt { get; set; }
public bool IsAutoGenerated { get; set; } = false; // AIが自動生成したか
public bool IsApproved { get; set; } = true;      // ユーザーが承認済みか
public string? SourceInteractionId { get; set; }  // 元になった会話ID
```

#### 新サービス: `SkillLearningService`

```csharp
// Services/SkillLearningService.cs
public class SkillLearningService
{
    // ─── フィードバックループのエントリーポイント ───
    
    /// タスク完了後に呼び出す。反省→スキル候補抽出→DB保存。
    public async Task LearnFromInteractionAsync(
        string task, string finalOutput, List<AgentStep> steps, int userId);
    
    /// スキル有効性の評価を更新する（Reviewerのスコアを基に）
    public async Task UpdateSkillMetricsAsync(
        int skillId, bool wasSuccessful);
    
    /// 低パフォーマンススキルを自動無効化し、改善提案を生成
    public async Task AuditAndRefineSkillsAsync(int userId);
    
    // ─── 内部実装 ───
    
    /// Reflection Agentがタスクを分析して新スキルを提案
    private async Task<List<SkillProposal>> ReflectAndProposeAsync(
        string task, string outcome, List<AgentStep> steps);
    
    /// 既存スキルのTriggerKeywordsを改善
    private async Task RefineExistingSkillAsync(Skill skill, List<string> recentInputs);
}

public record SkillProposal(
    string Name,
    string Description,
    string TriggerKeywords,
    string? BoundAgentRole,
    string Rationale      // なぜこのスキルが必要か
);
```

#### 学習フロー全体図

```
ユーザーがタスク入力
    ↓
CooperateAsync 実行
    ↓ (各ステップで)
SkillLearningService.UpdateSkillMetrics (使用したスキルを記録)
    ↓ (タスク完了後、fire-and-forget)
SkillLearningService.LearnFromInteraction
    ├── Reflection Agent呼び出し:
    │   "このタスクから将来再利用できるスキルパターンを抽出してください"
    │   → JSON: [{name, description, keywords, rationale}, ...]
    ├── 類似スキルがなければ IsApproved=false でDB保存（ペンディング）
    └── UIで「新しいスキル候補が見つかりました」通知

ユーザーがスキル管理画面で候補を確認・承認/却下
    ↓ (承認)
IsApproved = true → 次回から自動適用

定期監査 (SkillAuditTask, 週1回):
    低SuccessRate(<30%) スキルを無効化
    高SuccessRate(>80%) スキルのTriggerKeywordsをAIが拡充
```

#### Reflection Agent プロンプト

```
あなたはスキル抽出の専門家です。
以下のタスクの実行記録を分析し、将来の類似タスクで再利用できる
「スキルパターン」を抽出してください。

タスク: {task}
最終結果: {finalOutput}
エージェント実行ログ:
{steps.Select(s => $"[{s.Role}]: {s.Output[..Math.Min(200, s.Output.Length)]}...")}

以下のJSON形式のみで回答してください:
[
  {
    "name": "スキル名（英語CamelCase）",
    "description": "このスキルが何をするかの説明（AIへの指示として書く）",
    "triggerKeywords": "発火キーワード（カンマ区切り）",
    "boundAgentRole": "Executor|Reviewer|null",
    "rationale": "このスキルが必要な理由（1文）"
  }
]
抽出できない場合は [] を返してください。
```

### 3.3 GAP-9: UI改善

#### 現状の問題と解決策

**問題1**: サイドバーが情報過多  
**解決**: タブ化 + 折りたたみ
```
サイドバー構成:
[🗨️ チャット] [📁 プロジェクト] [🧩 スキル]
```

**問題2**: マルチエージェント協調の起動が不明確  
**解決**: チャット入力欄の上部に明確なモード切り替えを配置
```
[ 通常チャット | 🤝 協調モード ]
協調モード時: エージェント選択チェックボックスが展開表示
```

**問題3**: スキル候補の通知がない  
**解決**: 画面右下にトースト通知
```
💡 新しいスキル候補: "DocWriter" が発見されました
[確認する] [後で]
```

**問題4**: 初回ユーザーへのガイダンス不足  
**解決**: オンボーディングウィザード（3ステップ）
```
Step 1: プロバイダーを選択
Step 2: 最初のプロジェクトを作成（スキップ可）
Step 3: チャットを開始
```
