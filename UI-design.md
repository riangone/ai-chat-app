这是一份针对 AI 聊天界面的设计文档。本设计旨在平衡功能性与简洁感，核心架构参考了主流 AI 工具（如 ChatGPT, Claude, Gemini），并重点优化了**响应式布局**与**多模态交互**。
## 1. 设计目标
 * **多模态支持**：集成语音输入、文件附件、图片预览。
 * **极简主义**：减少视觉干扰，突出对话内容。
 * **跨设备适配**：在桌面端、平板和移动端均能提供原生级体验。
## 2. 界面布局架构 (Responsive Layout)
界面采用三栏/两栏式结构：
| 屏幕宽度 | 侧边栏 (Sidebar) | 聊天主容器 (Main Chat) | 详情面板 (Optional) |
|---|---|---|---|
| **Desktop** (>1024px) | 固定显示 (260px) | 居中流式布局 (max-width: 800px) | 默认隐藏或侧边弹出 |
| **Tablet** (768px-1024px) | 抽屉式 (通过图标唤起) | 全宽 | 不显示 |
| **Mobile** (<768px) | 隐藏 | 全宽 (优化输入框高度) | 不显示 |
### 响应式断点建议
 * **Max-width (Content)**: 对话内容区域建议限制在 800px 以内，以保证阅读时的视线扫描舒适度。
## 3. 核心组件设计
### A. 输入框容器 (Input Area)
这是 UI 的灵魂，建议使用“浮动框”或“底部固定条”设计。
 * **多行文本框**：支持自动增高，设置 max-height: 200px 超过后出现滚动条。
 * **左侧功能区**：
   * **附件按钮 (+)**：点击弹出文件选择或拖拽区。
 * **右侧功能区**：
   * **语音按钮 (Mic)**：长按或点击开启实时录音。
   * **发送按钮 (Send)**：未输入文本时可与语音按钮合并，输入后变为发送图标。
### B. 附件预览区 (Attachment Gallery)
位于输入框正上方。
 * **交互**：用户上传文件后，以小卡片形式展示（图标+文件名），右上角带删除 (X) 按钮。
 * **状态**：显示上传进度条（Progress Bar）。
### C. 消息流 (Message Feed)
 * **AI 消息**：浅色背景（如 #F7F7F8）或直接白底，搭配品牌头像。
 * **用户消息**：深色背景气泡或右对齐简洁文本。
 * **Markdown 支持**：必须支持代码高亮、表格渲染及公式渲染（LaTeX）。
## 4. 关键交互逻辑
### 语音交互 (Voice Interaction)
 1. **静默状态**：显示麦克风图标。
 2. **录音状态**：图标变为红色或伴随波纹动画（Waveform Animation）。
 3. **转文字**：在输入框内实时流式显示识别出的文字。
### 附件处理
 * **文件限制**：UI 应提示单文件大小限制（如 25MB）及支持格式（PDF, Docx, CSV, Images）。
 * **拖拽**：支持将文件直接拖入整个聊天窗口，触发“Drop Zone”蒙版。
## 5. 技术实现建议 (Tech Stack)
如果你打算动手实现，建议使用以下技术栈：
 * **框架**: React 或 Vue 3
 * **样式**: **Tailwind CSS** (极其适合响应式开发)
 * **UI 组件库**: **DaisyUI** 或 **Shadcn UI** (提供现成的 Input 和 Avatar 组件)
 * **图标库**: Lucide React 或 FontAwesome
 * **动画**: Framer Motion (用于处理侧边栏滑出和消息弹出动画)
## 6. 视觉规范 (UI Kit)
| 元素 | 建议样式 |
|---|---|
| **Primary Color** | #3B82F6 (科技蓝) 或 #10A37F (OpenAI 绿) |
| **Border Radius** | 12px (气泡), 24px (输入框) |
| **Font Stack** | Inter, system-ui, sans-serif |
| **Dark Mode** | 背景 #171717, 文本 #ECECF1 |

## 7. 接下来可以优化的点
 * **流式输出 (Streaming)**：打字机效果，增加交互的即时感。
 * **停止生成按钮**：AI 响应时，发送按钮转变为“停止”方块图标。
 * **引用功能**：支持对历史消息进行引用回复。
